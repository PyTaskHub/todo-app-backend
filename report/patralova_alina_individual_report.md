### 2.3 Backend-функциональность

#### Feature: Get Categories List (#17)

* Реализован `GET /api/categories` с авторизацией
* Возвращаются категории только текущего пользователя
* Учитывается количество задач в категории

**Результат:** Реализован базовый эндпоинт чтения категорий по требованиям.

---

#### Feature: Delete Category (#21)

* Реализован `DELETE /api/categories/{id}` с авторизацией
* При удалении категории у задач сбрасывается `category_id`
* Возвращается `204 No Content`

**Результат:** Безопасное удаление категорий без нарушения целостности данных.

---

#### Feature: Get Task Statistics (#47)

* Реализован `GET /api/tasks/stats` с авторизацией
* Подсчет total/completed/pending задач пользователя
* Расчет и корректная обработка `completion_rate`  

**Результат:** Пользователь получает агрегированную статистику по задачам.

---

#### Feature: Health Check Endpoint (#45)

* Реализован `GET /health` без авторизации
* Проверяется доступность БД (connection check)
* Возвращается статус приложения и timestamp
* Обработан сценарий недоступной БД (503)

**Результат:** Добавлен технический эндпоинт для мониторинга состояния сервиса.

## 4. Саморефлексия

Участие в проекте PyTaskHub стало для меня **первым опытом** выполнения задач в формате, приближенном к **реальной командной разработке**. Работа велась через доску задач с распределением ответственности и отслеживанием прогресса. GitHub выступал основной платформой для взаимодействия, согласования изменений и контроля версий. В ходе проекта освоила базовые практики командной работы с репозиторием: ориентирование в структуре проекта и модулях, работу с ветками, подготовку и внесение изменений, разрешение конфликтов и интеграцию правок через pull request. Также получила практический опыт написания и встраивания отдельных фрагментов кода в существующую архитектуру.

**Свой вклад** в проект оцениваю как комплексный и ориентированный на обеспечение целостности результата: принимала участие в анализе требований, подготовке проектной документации и ручной проверке корректности работы эндпоинтов. Параллельно выполняла отдельные задачи backend-разработки и тестирования: участвовала в реализации отдельных эндпоинтов и в подготовке тестов для модуля аутентификации. В ходе сверки документации с реализацией и тестирования API выявляла баги и инициировала корректировки, что способствовало повышению согласованности между требованиями, документацией и фактическим поведением системы. Такой формат участия позволил сформировать целостное понимание структуры проекта и при необходимости подключаться к задачам разных типов, поддерживая команду на смежных этапах разработки. Главным достижением считаю сформированную многопрофильность: возможность работать на стыке аналитики, документации, разработки и тестирования и обеспечивать связность результатов на каждом этапе.

Наиболее **трудоемким направлением** стала работа с документацией. На старте требовалось описывать требования и API при неполном понимании внутренних связей между сущностями и эндпоинтами. По мере погружения в кодовую базу и проверки поведения эндпоинтов сформировалось более системное представление об архитектуре, что упростило дальнейшую актуализацию документов. В процессе работы также фиксировались расхождения между документацией и реализацией, что показало важность регулярной сверки материалов с кодом и оперативного внесения правок для предотвращения накопления неточностей.

**Работа в команде** была организована эффективно и слаженно: участники поддерживали открытый и своевременный диалог, оперативно отвечали на вопросы и помогали в решении возникающих задач. Отдельно стоит отметить гибкость командного взаимодействия - в ситуациях, когда не было возможности присутствовать на синхронных созвонах, команда обеспечивала информирование и поддержку в асинхронном формате. Возможность открыто обсуждать возникающие сложности, получать помощь и не оставаться с проблемами один на один способствовала сохранению продуктивности и формированию положительной рабочей атмосферы. Полученный опыт командного взаимодействия считаю значимым для применения в дальнейшей профессиональной деятельности.
