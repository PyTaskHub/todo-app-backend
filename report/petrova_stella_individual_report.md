## 2. Перечень выполненных задач

### Backend-функциональность

#### Feature: Update User Profile (#19)

* Реализован `PUT /api/users/me` с авторизацией
* Реализована Pydantic схема `UserProfileUpdate`
* Создана и применена миграция добавленых в модель `User` полей `first_name` и `last_name`
* Проводится проверка уникальности email 
* Возвращает 409 если email уже занят другим пользователем (`email`, `first_name`, `last_name`)

**Результат:** Реализован базовый эндпоинт для обновления профиля текущего пользователя

---

#### Feature: Toggle Task Completion (#27)

* Реализованы `PATCH /api/tasks/{task_id}/` и `PATCH /api/tasks/{task_id}/uncomplete` с авторизацией
* Производится изменение `status` на "completed" или "pending"
* Возвращают 404 если задача не найдена или не принадлежит пользователю
* Возвращают 401 если нет токена (при авторизации)

**Результат:** Реализованы эндпоинты для отметки задачи как выполненной/невыполненной

---

#### Feature: Change Password (#43)

* Реализованы `POST /api/users/me/change-password` с авторизацией
* Реализована Pydantic схема `ChangePassword`
* Проводится проверка текущего пароля и хеширование нового
* Возвращает 401 если текущий пароль неверный
* Возвращает 422 для невалидного нового пароля

**Результат:** Реализованы эндпоинты для смены пароля (с проверкой текущего)

## 4. Саморефлексия

Участие в проекте PyTaskHub стало для меня первым опытом не только "настоящей" backend-разработки, но и выполнение задач в командной разработке. Работа велась через доску задач с распределением ответственности и отслеживанием прогресса. GitHub выступал основной платформой для взаимодействия, согласования изменений и контроля версий. Работа проходила следующим образом, где сначала создавалась отдельная ветка для своей фичи, после завершения разработки выполнялся push изменений и создавался pull request, который проходил код-ревью со стороны тимлида, после чего изменения сливались в основную ветку main. Получила важный и значительный практический опыт ориентировки, написания и встраивания отдельных фрагментов кода в существующую архитектуру.

**Моя работа в проекте** была связана с backend-задачами по написанию эндпоинтов и созданию схем при необходимости, а также изменение существующей модели и выполнение миграций. Во время выполнения задач уделяла особое внимание изучению существующих схем и эндпоинтов для реализации собственных по их подобию в соответствии с задачей. Подобная работа позволила примерно понять организацию архитектуры и работы в ней.

Наиболее **сложным этапом** работы было не написание эндпоинтов, с которыми были по началу трудности (в связи с отсутствием опыта), а с работой других технологий и инструментов. Возникли проблемы при настройке работы подключения необходимых модулей, докера и прочего для создания миграций и проверке корректности эндпоинтов через Swagger. Но главная проблема была при работе GIT-ом, где возникали ошибки и пропадали изменения в файлах, написанные локально на компьютере. Но проблемы решались и исправлялись в основном самостоятельно, где также отдельная благодарность команде при помощи в их решении.

**Работа в команде** была организована эффективно, где каждому своевременно оказывалась помощь и давались ответы по задачам. Отдельно хочу отметить организацию лидером проекта распределения по зоне ответственности подходящих задач, которые давали возможность работать с тем, что интересно и посильно для каждого члена команда. Также благодарна всем членам команды за полученный опыт в совместной работе над проектом, который позволил в удобной форме познакомиться с backend-разработкой и попробовать свои силы.
