# Daily Standup Guide

## Формат отчета
**Вчера:** Выполненные задачи  
**Сегодня:** Планируемые задачи  
**Блокеры:** Препятствия, требующие помощи команды

## Что такое блокер
Блокер — это препятствие, которое мешает выполнить задачу и не может быть решено самостоятельно.

## Примеры настоящих блокеров
- Ожидание code review от коллеги для продолжения работы
- Конфликт миграций с основной веткой
- Ошибка при запуске Docker / окружения разработки
- Отсутствие доступа к БД / серверу / документации
- Неясные требования в задаче, требующие уточнения
- Зависимость от незавершенной задачи другого разработчика
- Непонимание как реализовать техническое решение

## Что НЕ является блокером
- "Мало времени" / "Время" / "Не успеваю"
- "Работа" / "Учеба" / "Личные дела"
- "Сегодня не смогу, сделаю завтра"
- Болезнь (просто укажите в разделе "Сегодня")

## Если блокеров нет
Напишите: **"Блокеров нет"**, **"-"** или **"Отсутствуют"**

## Примеры

**Правильно:**
```
Вчера: Реализовал Task model, отправил PR #57
Сегодня: Исправлю замечания по ревью, начну Create Task endpoint
Блокеры: Ожидаю merge PR #56 (Category model) для настройки связей
```

**Неправильно:**
```
Вчера: Начал задачу
Сегодня: Продолжу вечером
Блокеры: Время, работа
```

**Правильно (без блокеров):**
```
Вчера: Завершил миграцию таблицы tasks
Сегодня: Напишу документацию к API, отправлю PR
Блокеры: Отсутствуют
```

## Цель раздела "Блокеры"
Своевременное выявление проблем для координации помощи от команды и предотвращения простоя в работе.
