## Отчёт по ручному тестированию с помощью Postman

### Сценарии для тестирования:

1. **Регистрация двух пользователей:**

Регистрация двух пользователей прошла успешно, ответ - 201 
 
2. **Login обоих пользователей:**

Login обоих пользователей прошел успешно, проверка - Get Current User отдает данные правильного пользователя.

3. **User 1: создает 3 задачи:**

User 1 успешно создает 3 задачи, в базе данных все 3 задачи видны, задачи отображаются правильно

4. **User 2: создает 2 задачи:**

User 2 успешно создает 2 задачи, в базе данных все 2 задачи видны, задачи отображаются правильно

5. **User 1: видит только свои 3 задачи:**

User 1 видит только свои 3 задачи - проверено с помощью эндроинта Get Tasks. Получили только задачи User 1.

6. **User 1: не может видеть задачи User 2:**

User 1 не видит задачи User 2

7. **User 1: обновляет свою задачу:**

Успешно обновлена задача User 1

8. **User 1 не может обновить задачу User 2:**

User 1 не может обновить не свою задачу - появляется ошибка 404 - Not found

9. **User 1: удаляет свою задачу:**

User 1 успешно удаляет свою задачу - запись в базе данных стирается

10. **Тесты валидации и ошибок:**

Эндопинт **"Register a new user"** - невалидные данные не принимает, ошибка 422. Проверяет уникальность email - ошибка 409.

Эндопинт **"Login user"** - невалидные данные не принимает, ошибка 422. Неверный логин или пароль - ошибка 401.

Эндпоинт **"Update User Profile"** - при попытке обновить профиль без авторизации ошибка 401 Unauthorized. Разрешенные данные обновляется, неразрешенные - нет. Если email уже зарегистрирован - ошибка 409. Невалидные данные - ошибка 422.

Эндпоинт **"Change Password"** - при попытке изменить пароль без авторизации ошибка 401 Unauthorized. При введении пустого нового пароля или менее 8 символов - ошибка 422. При введении неверного пароля, ошибка - 401. 

Эндпоинт **"Create New Task"** - при попытке создать новую задачу без авторизации ошибка 401 Unauthorized. Все поля проходит тестирование на валидацию данных, при неверных данных - ошибка 422. **При вводе в поле due_date просто чисел, дата обрабатывается в формате Unix Timestamp, в базу данных попадает дата 1970 года**

Эндпоинт "Get Tasks" - при попытке получить список задач без авторизации ошибка 401 Unauthorized.

Эндпоинт **"Update Task"** - при попытке обновить задачу без авторизации ошибка 401 Unauthorized, невозможно обновить не свою задачу - ошибка 404. Обновить можно поля вместе: "title", "description", "priority", "due_date". И каждый по отдельности. На невалидные данные - ошибка 422.

Эндпоинт "Get Single Task" - при попытке получить 1 задачу по ID без авторизации ошибка 401 Unauthorized.

Эндпоинт "Delete Task" - при попытке удалить задачу без авторизации ошибка 401 Unauthorized.

Эндпоинт "Get Task Statistics" - при попытке получить статистику по задачам без авторизации ошибка 401 Unauthorized.

Эндпоинт "Create Category" - при попытке создать категорию без авторизации ошибка 401 Unauthorized. Ответ на невалидные данные - ошибка 422.

Эндпоинт "Get User Categories" - при попытке получить категории пользовтаеля без авторизации ошибка 401 Unauthorized.

Эндпоинт "Update The Category" - при попытке обновить категорию без авторизации ошибка 401 Unauthorized. Ответ на невалидные данные - ошибка 422.

Эндпоинт "Delete Existing Category" - при попытке удалить категорию без авторизации ошибка 401 Unauthorized.

Эндпоинт "Health Check" - работает без авторизации

Эндпоинт "Root" - работает без авторизации
